Foundation · 01
What Is Lead Status?

A Contact property that tracks the sales team's day-to-day working state with a lead — from first contact to conversion or disqualification.

Core definition

Lead Status is a customisable dropdown property on Contact records that tells you what the sales team is currently doing with a lead. While Lifecycle Stage tells you the big-picture funnel milestone (which chapter), Lead Status tells you what's happening right now (which page). It is owned and managed by the sales team — reps update it as they work their leads.

Why the sales team needs it

Lead Status vs Lifecycle Stage

The most important distinction in HubSpot contact tracking. These two properties serve completely different purposes.

The analogy that makes it permanent

Lifecycle Stage = the chapter of the book. Lead Status = what's happening on the current page. A contact can be in the SQL chapter while their Lead Status is "In Progress" (rep is actively calling) or "Bad Timing" (spoken to them, right fit, not ready). The chapter doesn't change — only the page does. Never use Lead Status for funnel milestone reporting, and never use Lifecycle Stage for rep activity tracking.

System Logic

Five rules that govern how HubSpot handles Lead Status — know these before building any automation.

Any direction, any timeLead Status has no direction enforcement. New → Bad Timing → New → In Progress — all valid.No safety net. A careless rep can overwrite important statuses. Build required Reason properties to capture context before status changes.

Admin rule — less is more

Keep total Lead Status options under 12. More options = less adoption. If you have 18 options, reps either pick randomly or always pick the same 3. Run a quarterly usage report: any option with fewer than 5% of contacts assigned should be reviewed — either retrain the team on it or remove it.

Admin Setup · 06
Step-by-Step Setup
Path in HubSpot

Settings → Properties → Contacts → Search "Lead status" → Click to edit options

  1. Export current distribution — Before editing, check how many contacts are at each current option. Go to Contacts → filter by Lead Status = each value → note the counts. This prevents accidentally removing a status 300 contacts are using.
  2. Review default options — Decide which defaults to keep, which to rename, and which to remove. "Open" is commonly unused — if your team skips it, remove it to reduce noise.
  3. Add custom options — Click "Add option." Name it clearly. Set display order: active outreach options first (New, In Progress, Connected), then exit options last (Unqualified, Bad Timing). This ordering guides rep behaviour.
  4. Set a default value — In the property settings, set "New" as the default value. Every contact created manually or imported without a Lead Status will start at New automatically.
  5. Update MQL workflow — In your MQL promotion workflow, add: Set Lead Status = New. This ensures every MQL lands in the SDR queue as a fresh, unreviewed contact.
  6. Build Open Deal automation — Workflow: Deal is created AND associated Contact exists → Set Contact Lead Status = Open Deal. This removes a manual step reps consistently forget.
  7. Create the Reason property — Create a custom dropdown: "Lead Status Reason." See the Disqualification Reason section below for the full option list.
  8. Create Follow-up Date property — Custom Date picker property on Contact: "Lead Status Follow-up Date." Reps fill this when setting Bad Timing. The re-engagement workflow uses it as a trigger.
  9. Build the Bad Timing re-engagement workflow — This is the most valuable setup step. See the Workflows section for full details.
Admin Setup · 07
Disqualification Reason Property

The diagnostic data that improves your entire upstream process — most companies don't build this. Those that do see compounding returns.

Why this matters

Setting a contact to Unqualified or Bad Timing without capturing why is like a doctor discharging a patient without writing a diagnosis. You lose the data that would improve your lead generation targeting, your MQL criteria, and your content strategy. This one property, collected consistently over 6 months, drives more strategic improvement than almost any other admin initiative.

This feedback loop makes your entire demand generation engine smarter over time.

Admin Setup · 08
Key Workflows to Build

Four workflows that every portal with a sales team needs — in priority order.

Workflow 1 — MQL arrival (highest priority)
Enrolment trigger: Lifecycle Stage = Marketing Qualified Lead AND Lead Status is unknown (first time entering MQL)
  • Set Lead Status = New
  • Notify assigned SDR: "New MQL — {{contact.firstname}} at {{contact.company}}"
  • Create task for owner: "Review this MQL — due in 4 hours"
Workflow 2 — Open Deal auto-set
Enrolment trigger: Deal is created AND associated Contact is known
  • Set associated Contact Lead Status = Open Deal
Workflow 3 — Stale lead alert (rep accountability)
Enrolment trigger: Lead Status = New AND Create Date is more than 2 business days ago
  • Create urgent task for owner: "MQL sitting at New for 48h — review immediately"
  • Notify SDR manager via email
Workflow 4 — Bad Timing re-engagement (highest ROI)
Enrolment trigger: Lead Status = Bad Timing AND Lead Status Follow-up Date = today
  • Set Lead Status = New
  • Clear Lead Status Follow-up Date
  • Notify owner: "Time to re-engage — {{contact.firstname}} at {{contact.company}}. You marked them as Bad Timing on {{lead_status_follow_up_date_original}}."
  • Create task: "Re-engagement call — {{contact.firstname}}"
Build these in order

Workflow 1 is highest operational priority — every MQL must land in a rep's queue. Workflow 4 is highest ROI — it generates revenue from leads most teams forget. Build both in your first week of any new HubSpot setup.

Applied Learning · 11
Common Admin Mistakes
  • 01 — Using Lead Status for funnel reporting
    Lead Status = In Progress doesn't tell you about pipeline health — it tells you about rep workload. Reporting on Lead Status as a funnel metric confuses activity with progression. Use Lifecycle Stage for funnel analysis. Use Lead Status for rep activity analysis.
  • 02 — Not capturing disqualification reasons
    Setting Unqualified or Bad Timing with no reason loses your most valuable diagnostic data. Fix: build the Lead Status Reason property and make it required via a task workflow that fires on these transitions.
  • 03 — Letting In Progress become a parking lot
    Leads that stay In Progress for 30+ days are either being ignored or the rep is avoiding a difficult disqualification decision. Fix: build the 14-day stale In Progress alert workflow. Normalise honest disqualification.
  • 04 — Not automating Open Deal status
    Relying on reps to manually set Lead Status = Open Deal when they create a deal results in 30–50% of contacts never getting updated. The data becomes unreliable. Fix: build the workflow. Always.
  • 05 — Deleting a Lead Status option instead of archiving
    Deleting a Lead Status option removes the value from every contact who had it set — replacing their status with blank. Their Lead Status history is lost and any workflows referencing that option break silently. Fix: always archive, never delete.
  • 06 — Creating too many Lead Status options
    18 options means reps pick the first three they see or the same one every time. Data quality drops to near-zero. Fix: start with 8–10 options maximum. Quarterly review: remove options with less than 5% adoption.
  • 07 — Forgetting to build the Bad Timing loop
    Bad Timing contacts without a re-engagement system are forgotten pipeline. Every one of them was a qualified lead who just wasn't ready. Without automation, they never come back. Fix: Workflow 4 is non-negotiable. Build it in every HubSpot setup.

Admin Best Practices
  • 🔄
    Build the Bad Timing re-engagement loop as standard. Every HubSpot setup you do, this workflow goes in. It is the highest-ROI single automation in a sales CRM.
  • 📊
    Report on Lead Status transition velocity. How many days does a contact spend at New before moving to In Progress? Average days at In Progress before Connected? These benchmarks identify process bottlenecks and individual rep patterns worth coaching.

    Use disqualification data as a coaching tool. A rep with mostly "Wrong industry" in their Unqualified reasons is getting bad leads. A rep with mostly "Ghosted" is not running enough touchpoints. The data tells the coaching story.
  • 📋
    Align options with your actual sales motion. An inbound-only team may not need "Open" or even "New" the way an outbound SDR team does. Customise to reflect how your team actually works, not how an imaginary team works.
  • 🗓️
    Quarterly Lead Status audit. Review usage counts per option. Remove or retrain on options with less than 5% adoption. Review whether new options are needed based on rep feedback.
  • 🔔
    Make the SDR manager's alerts automatic. A weekly "stale MQL" summary email, auto-generated by HubSpot workflow, that shows the manager which reps have the most uncovered New contacts is a simple admin build with significant operational impact.

Interview Questions
"How does Lead Status differ from Lifecycle Stage, and when would you use each?"
Lifecycle Stage tracks the big-picture funnel milestone — it's shared across Marketing, Sales, and CS and drives all cross-functional reporting. Lead Status is a sales-specific working state — it tells a rep what action they're currently taking with a contact. Lifecycle Stage has seven fixed options and moves forward by default; Lead Status is fully customisable and can move any direction. The analogy: Lifecycle Stage is the chapter of the book, Lead Status is the current page. I'd report on pipeline health and MQL-to-customer conversion using Lifecycle Stage. I'd report on rep activity, disqualification patterns, and outreach velocity using Lead Status. They work together — neither replaces the other.
"What is the Bad Timing status and why do you consider it strategically important?"
Bad Timing means the contact is genuinely qualified — right fit, right seniority, real need — but the timing is wrong right now. Maybe they just signed a contract, are in a budget freeze, or are going through an acquisition. The strategic value is that these are warm leads who already know you. They will be ready in the future — and when that future arrives, a competitor could reach them first. My approach is to build a re-engagement workflow using a "Follow-up Date" custom property. When a rep sets Bad Timing, they fill the date. On that date, the workflow resets Lead Status to New and notifies the owner. Done consistently, Bad Timing contacts become one of the most reliable pipeline sources — I've seen this generate 15–25% of monthly new pipeline at companies with good CRM hygiene. Most teams leave this completely untapped.
"A sales manager says they have no visibility into what reps are doing with their MQLs. How do you solve it in HubSpot?"
First, I'd audit the current Lead Status setup: are options aligned with the actual sales process, are reps actually updating them, and is there a default set on new contacts? Then I'd build four things. One: an MQL arrival workflow that auto-sets Lead Status to New and creates a review task for the rep — so every MQL is acknowledged. Two: a stale New alert workflow that fires if a contact sits at New for more than 48 hours without being touched, notifying the manager. Three: a disqualification reason property so every Unqualified or Bad Timing contact has a logged reason — this turns "garbage leads" complaints into structured data. Four: a Lead Status activity dashboard showing contacts by status per rep, average days at New, and disqualification reason distribution. Within 30 days the manager would have full activity visibility and could make data-driven coaching decisions.
"You want to remove a Lead Status option that nobody seems to use. What's the right process?"
Never delete — always archive. Deleting a Lead Status option removes the value from every contact currently assigned to it, replacing their status with blank and breaking any active workflows or lists referencing that option. The right process: first, filter contacts by that Lead Status value and note the count. If there are contacts there, decide what their status should be updated to and bulk-update them before archiving. Check all active workflows for any branches or triggers that reference that option and update or remove them. Then archive the option in the property settings — it disappears from the dropdown but the historical data is preserved. Finally, document the change in your admin changelog so future admins understand why it was removed.

RevOps mindset on Lead Status

Lead Status is the activity data layer of your CRM. It enables three things Lifecycle Stage cannot: rep accountability reporting (who is working their queue?), disqualification analysis (why are leads failing, and what does that tell marketing?), and future pipeline prediction (Bad Timing contacts with known re-engagement dates = quantifiable projected revenue). Companies that build this system seriously — with custom options, disqualification reasons, and re-engagement automation — consistently outperform those that treat Lead Status as a simple dropdown they fill in manually. The difference is not the property itself — it's the system built around it.
